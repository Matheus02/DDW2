HTML 5

HTML 5 - criado pela W3C

O HTML 5 foi lançado em 2012, como um padrão a ser adotado pelos navegadores.

Ele visa a proporcionar suporte total a integração com CSS e javascript e eliminar a necessidade de plugins para muitas situaçoes comus, incorporando nativamente vários recursos que antes dependiam de um plugun. Foi desenvolvido para funcionar da mesma forma em diferentes dispositivos como PC, Tablet, Smartphone ou Smart TV.

O HTML 5 também cria novas tags e modifica a função de outras. As versões antigas do HTML não continham um padão universal para a criação e seções comuns e específicas como rodapé cabeçalho, sidebar, menus etc. Não havia um padrão de nomenclatura de IDs, Classes e Tags.

Ha outros elementos e atributos que sua função e significado foram modificados e que agora podem ser reutilizados de forma mais eficaz. Por exemplo, elementos descontinuados em versões anteriores do HTML, agora assumem funções diferentes e entregam mais significados aos usuários.

O doctype

O Doctype deve ser a primeira linha de código do documento antes da tag HTML. Ele indica para o navegador e para outros meios qual a especificação de código utilizar. Não é uma tag do HTML, mas uma instrução para que o browser tenha informações sobre qual versão de codigo a marcação foi escrita.

O elemento HTML

O código HTML, é uma série de elementos em árvores onde alguns elementos de outro e assim por diante. O elemento principal dessa árvore é sempre TAG HTML.

O atributo lang é necessário para que os user-agents saibam qual a linguagem principal do documento.

HEAD

A tag HEAD é onde fica toda a parte inteligente da página. No HEAD ficam os metadados.

Metatag Charset

É responsável por chavear qual tabela de caractere a página está utilizando.

Novos elementos e atributos

O HTML 5 trouxe um série de elementos que nos ajudam a definir setores principais no documento HTML. Com a ajuda destes elementos, podemos por exemplo diferenciar diretamente pelo código HTML 5 áreas importantes como sidebar, rodapé e cabeçalho. Conseguimos selecionar a área de conteúdo indicando onde exatamente é o texto do artigo.

Estas mudanças simplificam o trabalho de sistemas como os dos buscadores, assim eles conseguem vasculhar o código de maneira mais eficaz. Procurando e guardando informaçõe mais exatas elevando menos tempo para estocar informações.

Abaixo segue uma lista dos novos elementos e atributos

section A tag section define uma nova seção genérica no documento. Por exemplo, a home de um website pode ser dividida em diversas seções. Introdução ou destaque, novidades, informação do contato e chamadas para conteúdos.
nav O elemento nav representa uma seção da página que contém links para outras partes do website. Nem todos os grupos de links deve ser elementos nav, apenas aqueles que contenham links importantes. Isso pode ser aplicado naqueles blocos de links que geralmente são colocados no rodapé e também para compor o menu principal do site.
article O elemento article representa uma parte da página que poderá ser distribuído e reutilizável em FEEDs por exemplo. isto pode ser um post, artigo um bloco de comentário de usuário ou apenas um bloco de texto comum.
aside O elemento aside representa um bloco de conteúdo que referência o conteúdo que envolta do elemento aside. O aside pode ser utilizado para um bloco destinado a publicidade, ou até mesmo para definir um grupo de elementos nav.
hgroup Este elemento consite em um grupo de títulos. Ele serve para agruár elementos de título de h1 até h6, quando ele s tem múltiplos níveis como título com subtítulos e etc.
header Representa um grupo de introdução ou elementos de navegação. O elemento header pode ser utilizado para agrupar índices de conteúdos, campos de busca ou até mesmo logos.
footer Representa literalmente o rodapé da página. Seria o último elemento antes de fechar a tag HTML
timer Este elemento serve para marcar parte do texto que exibe um horário ou uma data precisa ao calensário gredoriano.
